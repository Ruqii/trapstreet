# Security Policy

## Reporting a vulnerability

Please open a private security advisory:
<https://github.com/AntiNoise-ai/trapstreet/security/advisories/new>

Or email **security@trapstreet.run**. We aim to respond within 72 hours.
Please do **not** open a public issue for suspected vulnerabilities.

## Supported versions

Trap Street is pre-1.0. Only the latest released version of each component
receives security fixes.

| Component               | Version | Supported |
| ----------------------- | ------- | --------- |
| `trapstreet-eval` skill | v0.1.x  | ✅        |
| `trapstreet-eval` skill | older   | ❌        |

## What the installer does

The `install.sh` script (served at <https://trapstreet.run/install.sh>, mirrored
from the [`trapstreet-eval-demo`](https://github.com/AntiNoise-ai/trapstreet-eval-demo)
repository) writes exactly three files to a single user-owned directory:

```
~/.claude/skills/trapstreet-eval/
├── SKILL.md
├── questions.json
└── grade.py            (chmod +x)
```

It does **not**:

- Run `sudo` or escalate privileges.
- Modify shell startup files (`.zshrc`, `.bashrc`, `.profile`, etc.).
- Install launch agents, login items, cron jobs, or daemons.
- Touch any path outside `~/.claude/skills/trapstreet-eval/`.
- Send telemetry, analytics, or any network request beyond fetching the files
  and their `SHA256SUMS` from `github.com`.
- Read or transmit user data, environment, or credentials.

To uninstall:

```sh
rm -rf ~/.claude/skills/trapstreet-eval
```

## Verifying a release

Every tagged release ships a `SHA256SUMS` file alongside the payloads:

```
https://github.com/AntiNoise-ai/trapstreet-eval-demo/releases/download/<TAG>/SHA256SUMS
```

`install.sh` fetches `SHA256SUMS` first and verifies every payload before
writing it to disk. To verify manually:

```sh
TAG=v0.1.0
BASE=https://github.com/AntiNoise-ai/trapstreet-eval-demo/releases/download/$TAG
curl -fsSLO "$BASE/SHA256SUMS"
for f in SKILL.md questions.json grade.py; do
  curl -fsSLO "$BASE/$f"
done
shasum -a 256 -c SHA256SUMS
```

A checksum mismatch means **do not install** — report it via the channels above.

## Trust model

When you run the installer, you trust:

1. **GitHub** as the release host and TLS endpoint.
2. **The maintainers' GitHub credentials.** We enforce hardware-key 2FA and
   protected branches; releases require code review.
3. **The contents of the pinned release tag** at the moment it was cut.

The installer does **not** require you to trust:

- The `trapstreet.run` domain beyond serving a static script that points at
  GitHub. The script's contents are reproducible from the source repo, and the
  recommended install paths let you inspect or bypass the domain entirely.
- Any future version of the script. The version is pinned explicitly; upgrades
  are opt-in by re-running the installer with a new `TRAPSTREET_VERSION`.
- Any third-party CDN, registrar, or DNS provider beyond GitHub's own.

## Trust-but-verify install path

We expect users to read what runs on their machine. The recommended path:

```sh
# 1. Download, don't pipe
curl -fsSL https://trapstreet.run/install.sh -o /tmp/trapstreet-install.sh

# 2. Read it
less /tmp/trapstreet-install.sh

# 3. Run it
bash /tmp/trapstreet-install.sh
```

Or, fully transparent — clone the repo and run from the checkout:

```sh
git clone https://github.com/AntiNoise-ai/trapstreet-eval-demo
cd trapstreet-eval-demo
./skill/install.sh
```

## What we do on our side

- Releases are cut from a protected `main` branch with required code review.
- Maintainer accounts have hardware-key 2FA enforced.
- `SHA256SUMS` files are generated by the release workflow and attached to the
  GitHub release alongside the payloads.
- We do **not** auto-update installed skills. Upgrades are opt-in: re-run the
  installer with the new `TRAPSTREET_VERSION`.
- The script pulls only from `github.com` over TLS 1.2+ with HTTPS enforced
  (`curl --proto '=https' --tlsv1.2`).
